אופרטורים חישוביים (Arithmetic Operators)

Java יודעת לעשות חשבון! האופרטורים הבסיסיים מוכרים, אבל יש כמה הפתעות — בעיקר סביב חילוק.
אופרטורפעולהדוגמהתוצאה
+חיבור7 + 310
-חיסור7 - 34
*כפל7 * 321
/חילוק7 / 32 (לא 2.33!)
%שארית (Modulo)7 % 31
⚠️ חילוק שלמים — הטעות #1 של מתחילים
כשמחלקים שני מספרים שלמים (int), Java זורקת את החלק העשרוני בלי לעגל!
7 / 3 = 2  |  5 / 2 = 2  |  1 / 3 = 0

פתרון: הפוך לפחות אחד מהם ל-double:
7.0 / 3 = 2.333...  |  (double) 7 / 3 = 2.333...
💡 מה זה שארית (%)?

שארית זה מה שנשאר אחרי חילוק שלם. 7 חלקי 3 = 2 שלמים, נשאר 1.
שימושים נפוצים: בדיקת זוגיות (n % 2 == 0), חלוקה ללא שארית, גלגול מונה.

Javaאופרטורים מקוצרים (Shorthand)
int x = 10;

x += 5;    // x = x + 5   → x = 15
x -= 3;    // x = x - 3   → x = 12
x *= 2;    // x = x * 2   → x = 24
x /= 4;    // x = x / 4   → x = 6
x %= 4;    // x = x % 4   → x = 2

// ++  ו  --  (הגדלה והקטנה ב-1)
x++;       // x = x + 1   → x = 3
x--;       // x = x - 1   → x = 2

סדר פעולות ו-Type Casting

בדיוק כמו במתמטיקה — כפל וחילוק לפני חיבור וחיסור. סוגריים תמיד ראשונים.
עדיפותאופרטוריםדוגמה
1 (גבוהה)( ) סוגריים(2 + 3) * 4 = 20
2++ -- (pre)++x
3* / %2 + 3 * 4 = 14
4 (נמוכה)+ -10 - 3 + 2 = 9
Type Casting — המרת סוגים:
🔄

המרה אוטומטית (Implicit)

מקטן לגדול — Java עושה לבד:
int → double
int x = 5; double y = x; // 5.0

✂️

המרה ידנית (Explicit Cast)

מגדול לקטן — חייבים לציין:
double → int
double y = 9.7; int x = (int) y; // 9

Javaדוגמאות Casting
// בעיה: חילוק שלמים
int a = 7, b = 2;
System.out.println(a / b);           // 3  (לא 3.5!)

// פתרון 1: casting
System.out.println((double) a / b);  // 3.5 ✓

// פתרון 2: ספרה עשרונית
System.out.println(7.0 / 2);          // 3.5 ✓

// casting חותך (לא מעגל!)
double grade = 92.8;
int rounded = (int) grade;            // 92 (לא 93!)
⚖️

אופרטורי השוואה (Comparison Operators)

אופרטורי השוואה משווים שני ערכים ומחזירים true או false. הם הבסיס לתנאים.
אופרטורמשמעותדוגמהתוצאה
==שווה ל5 == 5true
!=שונה מ5 != 3true
>גדול מ5 > 3true
>=גדול או שווה5 >= 5true
<קטן מ5 < 3false
<=קטן או שווה3 <= 5true
⚠️ == לעומת = (הטעות הכי נפוצה!)
= זה השמה (שמים ערך במשתנה)
== זה השוואה (בודקים אם שווה)

if (x = 5) ← שגיאה!
if (x == 5) ← נכון!

תנאים — if / else

תנאי if מאפשר לתוכנית לקבל החלטות. אם התנאי מתקיים (true) — מריצים קוד מסוים. אם לא — קוד אחר.
💡 אנלוגיה

חשוב על רמזור: אם ירוק → סע. אחרת → עצור. זה בדיוק if-else.

האם התנאי מתקיים?
✅ כן (true)
בצע את הקוד שב-if
❌ לא (false)
בצע את הקוד שב-else
Javaif בסיסי
int age = 20;

if (age >= 18) {
    System.out.println("אתה בגיר");
}
Javaif - else
int grade = 72;

if (grade >= 60) {
    System.out.println("עבר!");
} else {
    System.out.println("נכשל");
}
Javaelse if — שרשרת תנאים
int grade = 85;

if (grade >= 90) {
    System.out.println("מצוין!");
} else if (grade >= 80) {
    System.out.println("טוב מאוד");
} else if (grade >= 60) {
    System.out.println("עבר");
} else {
    System.out.println("נכשל");
}
// פלט: טוב מאוד
✅ טיפ חשוב
בשרשרת else-if, רק הבלוק הראשון שמתקיים ירוץ. למרות ש-85 גם גדול מ-60, הקוד עוצר אחרי "טוב מאוד".
Javaif מקונן (Nested if)
int age = 25;
boolean hasLicense = true;

if (age >= 17) {
    if (hasLicense) {
        System.out.println("מותר לנהוג");
    } else {
        System.out.println("צריך רישיון");
    }
} else {
    System.out.println("צעיר מדי");
}

אופרטורים לוגיים (Logical Operators)

מה אם יש כמה תנאים שצריכים להתקיים ביחד? או שמספיק שאחד יתקיים?
אופרטורשםמשמעותדוגמה
&&AND (וגם)שניהם חייבים trueage >= 18 && hasID
||OR (או)מספיק אחד trueisVIP || hasTicket
!NOT (לא)הופך true↔false!isClosed
Javaשימוש באופרטורים לוגיים
int age = 25;
boolean hasLicense = true;
boolean isDrunk = false;

// AND — שני התנאים חייבים
if (age >= 17 && hasLicense && !isDrunk) {
    System.out.println("מורשה לנהוג");
}

// OR — מספיק אחד
boolean isVIP = true;
boolean hasTicket = false;
if (isVIP || hasTicket) {
    System.out.println("כניסה מאושרת");
}
📝 Short-circuit evaluation
ב-&& אם הראשון false → לא בודקת את השני (ממילא false).
ב-|| אם הראשון true → לא בודקת את השני (ממילא true). יעיל ובטוח!
🔀

switch ואופרטור טרנרי

כשיש ערך אחד להשוות למספר אפשרויות, switch נקי יותר מ-if-else ארוך.
Javaswitch — יום בשבוע
int day = 3;

switch (day) {
    case 1: System.out.println("ראשון");  break;
    case 2: System.out.println("שני");    break;
    case 3: System.out.println("שלישי");  break;
    case 4: System.out.println("רביעי");  break;
    case 5: System.out.println("חמישי");  break;
    case 6: System.out.println("שישי");   break;
    case 7: System.out.println("שבת");    break;
    default: System.out.println("לא תקין");
}
// פלט: שלישי
⚠️ לא לשכוח break!
בלי break, Java ממשיכה לרוץ לכל ה-case הבאים (fall-through). בד"כ זו טעות.
אופרטור טרנרי (Ternary) — if-else בשורה אחת:
Javaאופרטור טרנרי
int age = 20;

// המבנה:  תנאי ? ערך_אם_true : ערך_אם_false
String status = (age >= 18) ? "בגיר" : "קטין";

System.out.println(status); // בגיר
✏️

תרגילים

✏️ תרגיל 1 — זוגי או אי-זוגי

קל

כתוב תוכנית שקולטת מספר שלם ומדפיסה אם הוא זוגי או אי-זוגי. רמז: שארית מחילוק ב-2.

Javaפתרון
import java.util.Scanner;

public class EvenOdd {
    public static void main(String[] args) {
        Scanner reader = new Scanner(System.in);
        System.out.print("הכנס מספר: ");
        int num = reader.nextInt();

        if (num % 2 == 0) {
            System.out.println(num + " הוא זוגי");
        } else {
            System.out.println(num + " הוא אי-זוגי");
        }
        reader.close();
    }
}

✏️ תרגיל 2 — מחשבון ציונים

בינוני

קלוט ציון (0-100) והדפס דרגה: 90+=מצוין | 80-89=טוב מאוד | 70-79=טוב | 60-69=מספיק | מתחת ל-60=נכשל. בונוס: בדוק שהציון תקין.

Javaפתרון
import java.util.Scanner;

public class GradeCalc {
    public static void main(String[] args) {
        Scanner reader = new Scanner(System.in);
        System.out.print("הכנס ציון (0-100): ");
        int grade = reader.nextInt();

        if (grade < 0 || grade > 100) {
            System.out.println("ציון לא תקין!");
        } else if (grade >= 90) {
            System.out.println("מצוין");
        } else if (grade >= 80) {
            System.out.println("טוב מאוד");
        } else if (grade >= 70) {
            System.out.println("טוב");
        } else if (grade >= 60) {
            System.out.println("מספיק");
        } else {
            System.out.println("נכשל");
        }
        reader.close();
    }
}

✏️ תרגיל 3 — יום בשבוע (switch)

קל

קלוט מספר 1-7 והדפס את שם היום. השתמש ב-switch. מספר לא תקין → הודעת שגיאה.

Javaפתרון
import java.util.Scanner;

public class DayOfWeek {
    public static void main(String[] args) {
        Scanner reader = new Scanner(System.in);
        System.out.print("הכנס מספר יום (1-7): ");
        int day = reader.nextInt();

        switch (day) {
            case 1: System.out.println("ראשון");  break;
            case 2: System.out.println("שני");    break;
            case 3: System.out.println("שלישי");  break;
            case 4: System.out.println("רביעי");  break;
            case 5: System.out.println("חמישי");  break;
            case 6: System.out.println("שישי");   break;
            case 7: System.out.println("שבת");    break;
            default: System.out.println("לא תקין!");
        }
        reader.close();
    }
}

✏️ תרגיל 4 — בודק כניסה למועדון

בינוני

גיל 18+ וגם תעודה → "כניסה מאושרת" | גיל 18+ בלי תעודה → "הבא תעודה" | מתחת ל-18 → "אסור לקטינים"

Javaפתרון
import java.util.Scanner;

public class ClubEntry {
    public static void main(String[] args) {
        Scanner reader = new Scanner(System.in);
        System.out.print("גיל: ");
        int age = reader.nextInt();
        System.out.print("יש תעודה? (true/false): ");
        boolean hasID = reader.nextBoolean();

        if (age >= 18 && hasID) {
            System.out.println("כניסה מאושרת");
        } else if (age >= 18) {
            System.out.println("הבא תעודה");
        } else {
            System.out.println("אסור לקטינים");
        }
        reader.close();
    }
}
