מהו Interface (ממשק)?

Interface הוא "חוזה" — הוא מגדיר מה מחלקה צריכה לדעת לעשות, בלי לומר איך. כל המתודות הן אבסטרקטיות כברירת מחדל.
💡 אנלוגיה

Interface = שלט רחוק. לא משנה אם זה טלוויזיה של סמסונג או LG — השלט מגדיר "יש כפתור הפעלה, כפתור ערוץ, כפתור ווליום". כל יצרן מממש אחרת.

Javaהגדרת Interface
public interface Printable {
    void print();         // אבסטרקטית אוטומטית
    String getDetails();  // אבסטרקטית אוטומטית
}

public interface Comparable {
    int compareTo(Object other);
}
🔌

implements — מימוש ממשק

Javaמימוש Interface
public class Invoice implements Printable {
    private int id;
    private double amount;

    public Invoice(int id, double amount) {
        this.id = id;
        this.amount = amount;
    }

    // חייבים לממש את כל המתודות!
    @Override
    public void print() {
        System.out.println("Invoice #" + id + ": $" + amount);
    }

    @Override
    public String getDetails() {
        return "Invoice{id=" + id + ", amount=" + amount + "}";
    }
}

// אפשר לממש כמה ממשקים!
public class Student implements Printable, Comparable {
    // חייב לממש את כל המתודות של שני הממשקים
}
⚠️ הבדל מהורשה
extends = הורשה, אפשר רק אב אחד
implements = מימוש ממשקים, אפשר כמה ממשקים!
זה הפתרון של Java לבעיית ההורשה המרובה.

פרויקט מסכם — מערכת ניהול חיות מחמד

פרויקט שמשלב את כל מה שלמדנו: מחלקות, הורשה, פולימורפיזם, ממשקים, מערכים.

🏆 פרויקט מסכם

מתקדם

בנו מערכת ניהול חנות חיות מחמד:

1. Interface: Feedable עם מתודות feed() ו-isHungry()

2. Abstract class: Animal עם שדות (name, age, type), בנאי, toString(), ומתודה אבסטרקטית speak()

3. מחלקות בנות: Dog, Cat, Bird — כל אחת מממשת speak() אחרת ו-implements Feedable

4. מחלקה PetShop עם מערך של Animal, ומתודות: addAnimal(), feedAll(), makeAllSpeak()

5. Main שיוצר חנות, מוסיף חיות, מאכיל את כולם ומדפיס את הקולות
